Nettoyage des reads

Ralisation du nettoyage

Nous allons nettoyer les fichiers FASTQ avec loutil fastp, qui process les fastq en fonction des rapports fastqc.

  • -l permet de filtrer les petits reads sur une taille minimum, ici 100 bp

  • -w permet dutiliser 8 threads

module load fastp/0.20.0
srun --cpus-per-task 8 fastp --in1 data/raw/SRR10390685_1.fastq --in2 data/raw/SRR10390685_2.fastq \
      -l 100  \
      --out1 data/outputs/SRR10390685_1.preprocessed.fastq --out2 data/outputs/SRR10390685_2.preprocessed.fastq \
      --unpaired1 data/outputs/SRR10390685_1.unpaired.fastq --unpaired2 data/outputs/SRR10390685_2.unpaired.fastq \
      -w 8 \
      -j data/reports/fastp.json -h  data/reports/fastp.html

Le rapport MultiQC est accessible ici.

Alignement

Puis on procde lalignement avec la fonction bwa mem. - -t permet dutiliser le multi-threading

  • samtools view permet de gnrer un format BAM standard (-b) avec entte (-h) et ignore la compatibilit avec des versions de Samtools antrieure (-S)

module load samtools/1.9

srun --cpus-per-task=33 bwa mem data/ref/GCF_000009045.1_ASM904v1_genomic.fna.gz data/outputs/SRR10390685_1.preprocessed.fastq data/outputs/SRR10390685_2.preprocessed.fastq -t 32 | samtools view -hbS > data/outputs/SRR10390685.bam

Extraction des reads chevauchant au moins 50% le gne trmNF.

Extraction du gff.gz

Un utilise gunzip pour extraire le fichier gff.


srun gunzip data/ref/GCF_000009045.1_ASM904v1_genomic.gff.gz

Extraction de la ligne du gff concernant le gne trmNF

Grace grep, on isole les lignes contenant les informations concernant le gne trmNF, puis grace awk, on slectionne la ligne qui contient les informations du gne (information dans la 3me colonne).


srun grep "trmNF" data/ref/GCF_000009045.1_ASM904v1_genomic.gff |\
 awk '$3=="gene"' > data/outputs/trmNF_gene.gff3

Sorting et indexing du bam

On trie le bam avec la fonction samtools sort, et on lindex avec la fonction samtools index.


srun samtools sort data/outputs/SRR10390685.bam -o data/outputs/SRR10390685.sorted.bam

srun samtools index data/outputs/SRR10390685.sorted.bam

Extraction des reads qui recouvrent trmNF > 50% de leur longueur

On extrait lintersect entre le bam tri et index et le gff, en utilisant bedtools intersect.

  • -f 0.50 permet de retenir les reads qui mappent au moins 50% de leur longueur sur le gne
module load bedtools/2.29.2

srun bedtools intersect -a data/outputs/SRR10390685.sorted.bam \
-b data/outputs/trmNF_gene.gff3 \
-f 0.50 >  data/outputs/SRR10390685.readsmappingtrmNF.bam

Puis pour compter le nombre de reads retenus, on utilise la fonction samtools view, avec loption -c qui permet de compter le nombre de reads.


srun samtools view -c data/outputs/SRR10390685.readsmappingtrmNF.bam

Il y a donc 2840 reads qui mappent trmNF sur au moins 50% de leur longueur.
